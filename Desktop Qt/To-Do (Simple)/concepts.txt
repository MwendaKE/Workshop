ERRORS(‚Äúvtable‚Äù error):
======================
This is a classic Qt ‚Äúvtable‚Äù error, and it‚Äôs because the Q_OBJECT macro in MainWindow wasn‚Äôt processed by Qt‚Äôs Meta-Object Compiler (moc).

In simple terms:
Whenever you use Q_OBJECT in a class (for signals and slots), Qt needs to generate extra code to handle signals/slots.
If you just compile with g++, that code isn‚Äôt generated, and you get undefined reference to vtable errors.

‚úÖ How to fix it
=================
Option 1: Use moc manually
---------------------------
Run the meta-object compiler on your header:
 $ moc mainwindow.h -o mainwindow.moc.cpp

Then compile everything including the generated file:
 $ g++ -fPIC main.cpp mainwindow.cpp mainwindow.moc.cpp -o todo `pkg-config --cflags --libs Qt5Widgets`

Run:
 $ ./todo

Option 2: Let qmake/make handle it (recommended for larger projects)
--------------------------------------------------------------------
Create a project file (todo.pro):

QT += widgets
SOURCES += main.cpp mainwindow.cpp
HEADERS += mainwindow.h

Generate Makefile:

qmake todo.pro
make

This automatically runs moc for any headers with Q_OBJECT.

Key takeaway:
------------
Whenever you use Q_OBJECT:
Either run moc manually and include the .moc.cpp in compilation
Or use qmake/CMake to handle it automatically
